## SOLUTIONS

### LEVEL 1 SOLUTIONS

#### Section 1A: Binary to Decimal

**1.1** 1011₂
= 1(8) + 0(4) + 1(2) + 1(1)
= 8 + 0 + 2 + 1 = **11** ✓ (Answer: B)

**1.2** 10000₂
= 1(16) + 0(8) + 0(4) + 0(2) + 0(1)
= 16 = **16** ✓ (Answer: B)

**1.3** 111₂
= 1(4) + 1(2) + 1(1)
= 4 + 2 + 1 = **7** ✓ (Answer: B)

**1.4** 1001₂
= 1(8) + 0(4) + 0(2) + 1(1)
= 8 + 0 + 0 + 1 = **9** ✓ (Answer: B)

**1.5** 11111₂
= 1(16) + 1(8) + 1(4) + 1(2) + 1(1)
= 16 + 8 + 4 + 2 + 1 = **31** ✓ (Answer: B)

---

#### Section 1B: Decimal to Binary

**1.6** 7₁₀
7 ÷ 2 = 3 R 1
3 ÷ 2 = 1 R 1
1 ÷ 2 = 0 R 1
Answer: **111** ✓ (Answer: C)

**1.7** 20₁₀
20 ÷ 2 = 10 R 0
10 ÷ 2 = 5 R 0
5 ÷ 2 = 2 R 1
2 ÷ 2 = 1 R 0
1 ÷ 2 = 0 R 1
Answer: **10100** ✓ (Answer: B)

**1.8** 15₁₀
15 ÷ 2 = 7 R 1
7 ÷ 2 = 3 R 1
3 ÷ 2 = 1 R 1
1 ÷ 2 = 0 R 1
Answer: **1111** ✓ (Answer: C)

**1.9** 50₁₀
50 ÷ 2 = 25 R 0
25 ÷ 2 = 12 R 1
12 ÷ 2 = 6 R 0
6 ÷ 2 = 3 R 0
3 ÷ 2 = 1 R 1
1 ÷ 2 = 0 R 1
Answer: **110010** ✓ (Answer: B)

**1.10** 100₁₀
100 ÷ 2 = 50 R 0
50 ÷ 2 = 25 R 0

6 ÷ 2 = 3 R 0
3 ÷ 2 = 1 R 1

### LEVEL 2 SOLUTIONS

#### Section 4: Binary ↔ Octal

**4.1** Convert 101110₂ to octal
Group by 3 from right: 101 | 110
= 5₈ | 6₈
**Answer: 56₈**
Verification: 5(8) + 6(1) = 46₁₀; Binary: 101110 = 32+8+4+2 = 46₁₀ ✓

**4.2** Convert 1010101₂ to octal
Group by 3 from right: 1 | 010 | 101
= 1₈ | 2₈ | 5₈
**Answer: 125₈**
Verification: 1(64) + 2(8) + 5(1) = 64+16+5 = 85₁₀
Binary: 1010101 = 64+16+4+1 = 85₁₀ ✓

**4.3** Convert 111000111₂ to octal
Group by 3 from right: 111 | 000 | 111
= 7₈ | 0₈ | 7₈
**Answer: 707₈**
Verification: 7(64) + 0(8) + 7(1) = 448+7 = 455₁₀
Binary: 111000111 = 256+128+64+4+2+1 = 455₁₀ ✓

**4.4** Convert 56₈ to binary
5₈ = 101₂
6₈ = 110₂
**Answer: 101110₂**

**4.5** Convert 123₈ to binary

**Answer: 001010011₂ or 1010011₂**

**4.6** Convert 777₈ to binary

**Answer: 111111111₂** (9 ones)

#### Section 7: Multi-base Conversion

**7.1** Convert 500₁₀ to binary, octal, and hexadecimal

To Binary:
500 ÷ 2 = 250 R 0
250 ÷ 2 = 125 R 0
125 ÷ 2 = 62 R 1
62 ÷ 2 = 31 R 0
31 ÷ 2 = 15 R 1
15 ÷ 2 = 7 R 1
7 ÷ 2 = 3 R 1
3 ÷ 2 = 1 R 1
1 ÷ 2 = 0 R 1
**Binary: 111110100₂**

To Octal:
500 ÷ 8 = 62 R 4
62 ÷ 8 = 7 R 6
7 ÷ 8 = 0 R 7
**Octal: 764₈**

To Hexadecimal:
500 ÷ 16 = 31 R 4
31 ÷ 16 = 1 R 15
1 ÷ 16 = 0 R 1
**Hexadecimal: 1F4₁₆**

**Answer: 500₁₀ = 111110100₂ = 764₈ = 1F4₁₆**

**7.2** Convert 1024₁₀ to all three bases

To Binary: 10000000000₂ (1 followed by 10 zeros)
To Octal: 2000₈
To Hexadecimal: 400₁₆

**Answer: 1024₁₀ = 10000000000₂ = 2000₈ = 400₁₆**

**7.3** Convert 11001100₂ to all bases

Decimal: = 128 + 64 + 8 + 4 = 204₁₀
Octal: Group by 3: 011 001 100 = 314₈
Hexadecimal: Group by 4: 1100 1100 = CC₁₆

**Answer: 11001100₂ = 204₁₀ = 314₈ = CC₁₆**

**7.4** Convert 2AB₁₆ to all bases

Decimal: = 2(256) + 10(16) + 11(1) = 512 + 160 + 11 = 683₁₀
Binary: 2=0010, A=1010, B=1011 = 001010101011₂
Octal: Group binary by 3: 001 010 101 011 = 1253₈

**Answer: 2AB₁₆ = 683₁₀ = 1010101011₂ = 1253₈**

#### Section 10: Problem Solving Answers

**10.1** 255 in binary is 11111111₂
This represents the maximum value for an 8-bit unsigned integer (maximum byte value).

**10.2** Subnet mask 255.255.255.0
255₁₀ = 11111111₂
0₁₀ = 00000000₂
So: 11111111.11111111.11111111.00000000

**10.3** Hash ABCD1234 (8 hex digits)
Each hex digit = 4 bits
8 × 4 = **32 bits**

**10.4** With 8 bits:
Maximum value = 2⁸ = 256 unique values (0 to 255)
